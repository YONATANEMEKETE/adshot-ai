# AdShot AI - Product Requirement Document (PRD)

**Last Updated:** 2026-04-11

## 1. Product Overview

**Product Name:** AdShot AI  
**Vision:** A high-end AI "Virtual Studio" that transforms flat product photos into professional lifestyle marketing assets in seconds.  
**Target Audience:**
- **E-commerce entrepreneurs:** Shopify/Etsy sellers who cannot afford professional photographers.
- **Social media managers:** Users who need fresh content for daily posts using the same product.

## 2. Core Features (MVP)

1. **Smart Background Removal:** Automatically isolates the product from the original image.
2. **Scene Generation (Thematic):**
   - **Themes:** Minimal, Natural, Urban, Seasonal
   - **Scene Richness:** Minimal, Balanced, Rich (controls environmental detail)
3. **Prompt Assist / Vibe Context:**
   - Users enter optional context or "vibe" (e.g., "coffee shop morning")
   - Rich prompts are generated by combining all user configurations via PromptLayer
4. **Orientation / Output Control:** Square, Portrait, Landscape (aligns output for specific social formats)
5. **Image Upscaling:** Ensures final output is sharp and social media-ready
6. **Gallery / My Shots:** Users can view, retrieve, and reuse past generations
7. **Download Options:** Download each variation individually or all at once
8. **Authentication:** Google OAuth sign-up/sign-in powered by Better Auth
9. **Pricing, Plans, and Usage Controls:**
   - Two primary plans at launch: **Free** and **Pro**
   - Users choose a plan as part of initial authentication/onboarding
   - Users can upgrade their plan later from the product
   - Plan entitlements determine generation limits, rate limits, export options, and other usage controls
   - Current launch positioning from the landing page:
     - **Free:** 5 AI-generated shots/month, all themes, basic background removal, standard-resolution downloads, public gallery access, no upscaling, no bulk download, no BYOK support
     - **Pro:** 100 AI-generated shots/month, priority generation, HD upscaling, bulk download, commercial usage support, visibility controls, BYOK support, early access to new themes/features
   - Exact quotas, perks, and enforcement details will be finalized in a dedicated pricing/limits spec
10. **Bring Your Own Key (BYOK):**
   - Users can connect their own model/API key and run generations against their own provider account
   - BYOK should reduce dependency on platform-managed credits for eligible workflows
   - Initial implementation details, supported providers, and guardrails will be finalized separately

## 3. Pages and Purposes (Updated)

1. **Landing Page**
   - **Purpose:** Convert visitors into active users by clearly communicating the product's value and capabilities.
   - **Content:** High-impact hero section, dynamic previews of AI-generated outputs, feature highlights, pricing tiers, and clear CTAs. Content structure is flexible and designed to scale as the product evolves.
2. **Public Gallery Page (New)**
   - **Purpose:** Inspire users and showcase community creativity by displaying AI-generated ad photoshoots.
   - **Content:** A browsable, grid-based showcase of all user-generated shots marked as public. Includes filtering by theme, style, or date, and quick-view options to explore how others are using the tool.
3. **Authentication Page**
   - **Purpose:** Provide secure, frictionless access to the platform.
   - **Content:** A single, unified authentication screen powered by Better Auth, exclusively supporting Google OAuth for both sign-up and login. No traditional email/password flows.
4. **Plan Selection / Onboarding Flow**
   - **Purpose:** Move newly authenticated users into an explicit plan before they begin generating assets.
   - **Content:** Clear plan comparison between Free and Pro, selection during onboarding, entitlement summary, and a path into the dashboard after plan confirmation.
5. **Dashboard (Virtual Studio)**
   - **Purpose:** The core creation workspace where users transform product photos into marketing assets.
   - **Content:** Drag-and-drop upload zone, configuration sidebar (Theme, Richness, Vibe Context, Orientation), prompt assist, and a real-time generation staging area with progress feedback.
6. **My Shots Page**
   - **Purpose:** Personal asset management, iteration, and retrieval.
   - **Content:** A private, grid-based library of the current user's generated shots. Mirrors the public gallery layout but includes user-specific features:
     - Version control & iteration history
     - Visibility toggles (public/private)
     - Bulk download & upscaling
     - "Remix/reuse" options for regenerating variations
7. **Settings / Account Page**
   - **Purpose:** Central hub for subscription, billing, API management, account controls, and other user-managed product preferences expected in an MVP SaaS product.
   - **Content:**
     - Current plan, plan comparison, upgrade/downgrade controls
     - Credit usage, generation usage, and rate-limit visibility
     - Billing history and subscription management
     - Profile details & preferences
     - Authentication/session controls and standard account management actions
     - **Bring Your Own Key (BYOK):** Allows users to connect external AI model API keys for advanced or custom generation workflows
     - Provider configuration, key status, and future model/provider settings

## 4. Tech Stack (Updated 2026)

### Frontend & Core Framework

- Next.js 16 (App Router, Turbopack enabled)
- React 19 (utilizing Actions API and `useOptimistic` for instant UI feedback)
- Tailwind CSS
- shadcn/ui (Radix UI primitives)
- Framer Motion (layout transitions and generation progress bars)

### State & Data Fetching

- Server state: TanStack Query (React Query) for caching and background revalidation
- Global state: Zustand (lightweight store for user preferences and "Virtual Studio" settings)

### Backend & Infrastructure

- Supabase (PostgreSQL, Realtime for generation updates)
- Better Auth (secure, developer-friendly authentication suite)
- Cloudinary (hosting original and generated high-res assets)
- Vercel (Edge Functions for low-latency AI orchestration)
- Polar or Stripe-backed subscription state, entitlements, and billing workflows

### AI Operations & Payments

- PromptLayer (prompt version control, A/B testing, cost tracking)
- Polar or Stripe (Merchant of Record for global tax compliance and subscriptions)
- PostHog (session replays and event tracking) or Plausible (privacy-first analytics)

## 5. User Flow

1. Landing Page: user explores value prop, pricing, and plan differences
2. Authentication: user signs up/in via Google OAuth (Better Auth)
3. Plan Selection: user selects Free or Pro during onboarding
4. Dashboard Entry: user lands on "My Shots" or starts a "New Shot"
5. Virtual Studio:
   - Upload original photo (immediate Cloudinary upload + Supabase record)
   - Select theme (Natural, Urban, etc.)
   - Adjust richness & orientation
   - Add "Vibe Context" (optional)
6. Entitlement Check: system verifies plan entitlements, rate limits, and available generation allowance before execution
7. Generation: click "Generate" -> triggers Vercel Edge Function
8. Review: view variations -> upscale, download, or mark as Public for Gallery based on plan capabilities
9. Post-generation: shot saved to "My Shots"; user can revisit, remix, share, or upgrade plan if limits are reached
10. Settings: user manages subscription, billing, BYOK, profile, and account preferences

## 6. Generation Pipeline Logic

1. Ingestion: user uploads image; system creates optimistic preview
2. Isolation: background removal via specialized API (ControlNet / Remove.bg)
3. Orchestration: app sends user config + isolated image to PromptLayer
4. Entitlement Resolution: app checks user plan, available quota, rate limits, and whether BYOK is enabled for the request
5. PromptLayer Resolution: pulls the latest production-ready prompt template and injects user variables
6. Generation: PromptLayer forwards request to the selected image model/provider (platform-managed or BYOK-backed)
7. Storage: output saved to Cloudinary; URLs stored in Supabase `shots` table
8. Delivery: UI updates via Supabase Realtime / query invalidation

## 7. Success Metrics

- **Visual Fidelity:** Realistic shadows and seamless scene integration
- **Speed:** Generation-to-display time under 15 seconds
- **UX Quality:** Minimalist interface with no "flash of unstyled content"
- **Global Readiness:** Successful payment processing in multiple currencies via Polar/Stripe
- **Community Engagement:** % of users marking shots as public; gallery interaction rates
- **Subscription Conversion:** % of authenticated users selecting a plan and upgrading from Free to Pro
- **Usage Governance:** Reliable enforcement of plan-based quotas, rate limits, and BYOK rules without blocking valid generations

## 8. MVP Goals

- Secure authentication and user data isolation (Google OAuth via Better Auth)
- Working "Virtual Studio" pipeline for single-product generation
- Functional "My Shots" gallery with filtering by theme + visibility controls
- Public Gallery page with community showcase & moderation safeguards
- Integrated plan selection, subscription, and usage-limit system for Free and Pro
- Settings page that handles billing, plan management, BYOK, profile controls, and core account preferences
- BYOK (Bring Your Own Key) foundation in Settings for future extensibility
